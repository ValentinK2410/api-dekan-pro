# Исправление дублирования UI при появлении второго игрока

## Проблема
При появлении второго игрока на экране дублируются надпись и кнопка (иконка «бегущий человек» на правой панели).

## Где искать в Unity (MultiplayerPersona)

1. **Скрипты создания кнопок справа**
   - Поиск по: `Button`, `AddPlayerButton`, `PlayerButton`, правой панели
   - Логика «создать кнопку при добавлении игрока» — вероятная причина дубликатов

2. **RemotePlayersManager**
   - Создание/обновление представлений удалённых игроков
   - Проверить: создаётся ли UI-элемент для каждого игрока из API и не дублируется ли он

3. **RemotePlayerView или префаб удалённого игрока**
   - Содержит label (имя) и кнопку
   - Если префаб инстанциируется несколько раз для одного игрока — будет дублирование

4. **Локальный vs удалённый игрок**
   - Локальный игрок: одна кнопка «бегущий человек» (своя)
   - Удалённый игрок: должна быть одна кнопка на игрока, не дублироваться
   - Убедиться, что локальный игрок не попадает в список «других» и не получает вторую кнопку

## Типичные исправления

### Вариант A: Кнопка создаётся на каждого игрока
```csharp
// ПЛОХО: создаём кнопку при каждом обновлении списка
foreach (var player in players) {
    var btn = Instantiate(playerButtonPrefab, panel);
    btn.SetPlayer(player);  // если players включает локального — дубль
}

// ХОРОШО: создаём только для удалённых, по user_id
foreach (var player in players) {
    if (player.user_id == myUserId) continue;  // пропустить себя
    if (existingButtons.ContainsKey(player.user_id)) {
        existingButtons[player.user_id].Update(player);
    } else {
        var btn = Instantiate(...);
        existingButtons[player.user_id] = btn;
    }
}
```

### Вариант B: Одна статическая кнопка «бегущий человек»
Если кнопка одна на всю игру (не на каждого игрока), дублирование возможно из‑за:
- Двойного вызова `Instantiate` для одной и той же UI
- Префаба в сцене + создание копии скриптом

**Решение:** Убедиться, что кнопка создаётся один раз (в Awake/Start) или берётся по `GetComponent`/ссылка в Inspector.

### Вариант C: Canvas / World Space UI
Если label и кнопка — World Space над головой:
- Привязать к одному игроку и не создавать заново при каждом poll
- Использовать object pool или проверку «уже создан для этого user_id»

## Как проверить
1. Открыть Unity-проект MultiplayerPersona в Cursor
2. Найти скрипты правой панели и RemotePlayer* 
3. Внести правки по одному из вариантов выше
